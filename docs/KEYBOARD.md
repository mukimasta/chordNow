# ChordNow 键盘与和弦修饰（产品设计）

下文为**目标交互与和声语义**的约定；运行中的代码若尚未对齐，以后续实现与 `CHANGELOG` 为准。

**物理键 `.` 与 `/` 与解析器内部字段交叉映射**：**`.`** → 属七类（`HarmonyModifiers.slash`）；**/** → 调内七和弦类（`HarmonyModifiers.period`）。

---

## 1. 级数


| 键位      | 含义                        |
| ------- | ------------------------- |
| `1`–`7` | **I**–**VII** 级（大调罗马数字级数） |
| `8`     | 与 `1` 同为 **I** 级          |


---

## 2. 核心修饰（层与语义）

### 2.1 `Shift`（左或右）——**大小和弦转换**

- 在**三和弦**框架上，在**同根**的大三与小三之间切换（或等价语义：把当前级数上的大/小性质翻转）。
- 与其它修饰组合时的先后顺序、与级数同时按下的规则，在实现里写死；文档层只约定「Shift = 大小转换」这一条。

### 2.2 `.`（`Period`）——**属七与其它组合**

- **单独按 `.`**（配合级数）：当前级数根音上的**属七和弦**（大小七，`dominant7`；如 **C7**、**G7**）。
- **`n` + `.`** → **减七和弦**。
- **V 级 + Shift + `.`** → **小七**（m7）。
- 与 **`/`**（调内七）同时按下时，以实现中的优先级为准（通常 **`.` 的属七**优先）。

### 2.3 `/`（`Slash`）——**调内七和弦**

- 在当前级数对应的**有效三和弦**上叠七度，得到**调内七和弦**；具体为 maj7 / m7 / V 属七 / ø 等，由**调内规则 + 级数**决定。
- **`/` + `9`** → 九和弦（在调内七和弦框架上叠九度）。
- 与 `Shift` 分属不同层，可组合。

### 2.4 `,`（`Comma`）——**六和弦**

- 表示在当前和弦结构上得到**六和弦**（大六 / 小六等，由调内性质与根音决定；如三和弦加六度 **6**）。

### 2.5 `9`——**add9**

- **`9`**（`Digit9`）：在当前和弦结构上增加九度音，调内大九/小九依和弦性质由解析规则决定。

### 2.6 `'` / `;`——**sus**


| 键位  | `code`（典型）  | 语义                         |
| --- | ----------- | -------------------------- |
| `'` | `Quote`     | **sus4**（挂四；三度换四度等，与解析器一致） |
| `;` | `Semicolon` | **sus2**（挂二）               |


与 `Shift`、`.`、`/`、`,`、`9`、`'`、`;`、`n`、`m` 同时按下时的允许组合与优先级：**首版建议**在实现里禁止未经验证的组合，或按「先结构和弦类型，再叠延伸音 / sus」的顺序解析。

与 `o` / `p` 同时按下时，sus 与转位的先后顺序在实现里写死。

### 2.7 `n` / `m`——**减和弦 / 增和弦**


| 键位  | `code`（典型） | 语义                                      |
| --- | ---------- | --------------------------------------- |
| `n` | `KeyN`     | **减和弦**（减三和弦及同类，相对当前级数根音；具体音集与标签与解析器一致） |
| `m` | `KeyM`     | **增和弦**（增三和弦及同类）                        |


- **`n` + `/`（无 `.`）** 等组合见 `harmony.ts`（减三和弦上叠七度等）。
- **`n` + `.`** → **减七和弦**（与 §2.2 一致）。

---

## 3. `o` / `p`：转位（低音位）


| 键位  | `code`（典型） | 含义                                 |
| --- | ---------- | ---------------------------------- |
| `o` | `KeyO`     | **第一转位**：**三音**在最低声部（相对该和弦根音的转位关系） |
| `p` | `KeyP`     | **第二转位**：**五音**在最低声部               |


- **和声语义不变**：罗马数字级数、和弦类型（三和弦 / 七和弦等）仍按级数与 `Shift`、`.`、`/`、`,`、`9`、`'`、`;`、`n`、`m` 等规则解析；**`o` / `p` 只改变「哪一个和弦音落在最低音」**，不改变根音级与和弦符号所指的类型。
- **声部实现约定**：转位时**只变换最低音**到对应和弦音（三音或五音在 bass）；**上层各音的 voicing、与上一和弦之间的就近连接、开放排列等规则与未按转位时相同**，不因转位而改用另一套上层逻辑。

同时按住 `o` 与 `p` 时的优先级（例如后按者优先）在实现里写死。

---

## 4. 其它键位（规划）

- **钢琴式一行隐喻** `1 q 2 w 3 4 r 5 t 6 y 7 8`：黑白键布局规划，与数字级数可叠合或再映射，落地时另述。

---

## 5. 小结表


| 键或组合              | 产品设计语义                                      |
| ----------------- | ------------------------------------------- |
| `Shift`           | 大小和弦转换（同根大↔小）                               |
| `.`               | **属七**（大小七）；与 `Shift` 在 V 上 → **小七**；与 `n` → **减七** |
| `/`               | **调内七和弦**（maj7 / m7 / …）                     |
| `,`               | **六和弦**                                     |
| V + `Shift` + `.` | **小七**（m7）                                  |
| `n` + `.`         | **减七**（dim7 等）                              |
| `n`               | **减和弦**                                     |
| `m`               | **增和弦**                                     |
| `9`               | add9                                        |
| `'`               | sus4                                        |
| `;`               | sus2                                        |
| `o`               | 三音在低音（第一转位）；**仅**改最低音，上层 voicing 连接规则不变     |
| `p`               | 五音在低音（第二转位）；同上                              |


---

## 6. 实现与文档

- 键位监听以 `KeyboardEvent.code` 为准（如 `Period`、`Slash`、`Quote`、`Semicolon`、`ShiftLeft` / `ShiftRight`、`Digit9`、`KeyN`、`KeyM`、`KeyO`、`KeyP`）。
- 表单控件聚焦时不抢和弦键等行为，仍由 `chordKeyboard` 一类逻辑约束。
